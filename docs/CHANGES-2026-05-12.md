## 三、新增命令

### `goct find`（govc 风格统一查找）

```
goct find -type m -name web              # 所有名字含 web 的 VM
goct find -type h --cluster c1           # 集群 c1 内所有主机
goct find -name prod                     # 跨所有类型查找 prod
goct find -type m --id-only | xargs -I{} goct vm.power.on {}
```

类型短名与 govc 兼容：`m`=VM、`h`=Host、`c`=Cluster、`d`=Datastore、`n`=Network；
CloudTower 独有的补充：`v`=VLAN、`f`=Folder、`g`=PlacementGroup、`t`=Template、`l`=Label、`u`=User、`a`=Alert。

输出统一 `TYPE / ID / NAME` 三列；支持 `--id-only` 与 `--json`。
单类型失败不会终止整个 find（仅在 stderr 打 warning），适合一键巡检。

### `goct events`（基于 user_audit_log）

```
goct events                             # 最近 50 条事件
goct events --user admin --action create_vm
goct events vm-uuid                     # 某资源的事件
goct events --follow                    # tail，每 2s 拉一次
goct events --json                      # NDJSON 输出（适合 jq）
```

CloudTower 没有"事件"概念，最接近的是 user_audit_log（用户审计日志）。
`pkg/adapter/event.go` 新增 `EventOps` 接口包装该 SDK，cmd 层提供 govc events 风格 UI。

### `goct vm.ip`（govc 风格增强）

之前只是把 `VM.IPs` 全部换行打印；现在对齐 govc：

```
goct vm.ip myvm                  # 第一个 IP，等待 VM tools 上报（默认超时 5m）
goct vm.ip myvm -a               # 全部 IP
goct vm.ip myvm --v4             # 只输出 IPv4
goct vm.ip myvm --v6             # 只输出 IPv6
goct vm.ip myvm --no-wait        # 立即读，无 IP 则空输出
goct vm.ip myvm --wait 30s       # 自定义超时
```

CloudTower 没有 vSphere 的 push 事件，客户端用 2s 间隔轮询 `GetVM`，
拿到第一个 IP 即返回（与 govc 行为一致）。

## 五、行为兼容提示

- `vm.create` 不再有默认磁盘和默认网卡 —— 已经依赖 `goct vm.create --name x --cluster y` 然后期待"自动给我一块 10G 盘"的脚本会在 CloudTower 端报错。请改为显式 `--disk size=10g --nic vlan=<id>`。
- `vm.disk.update` 现在要求 `--vm`，否则报错；但本来之前的实现就是个 no-op（什么都不改），所以等于第一次能用。
- `vm.migrate` 不指定 `--host` 时由 CloudTower 自动选目标主机；如果之前依赖 goct 在客户端 random 选 host，请显式传 `--host`。
- `vm.nic.update` 现在要求 `--vm` 和 `--nic-id`（之前没传 vm 也能调，但调用结果是 server 端报错）。
- `task.ErrFailed` 还在，但实际是 `adapter.ErrTaskFailed` 的别名，调用方代码无需改。
